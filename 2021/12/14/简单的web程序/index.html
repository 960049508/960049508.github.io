简单的web程序

发表于2021-12-14|更新于2021-12-15|实验
实验八 实现一个简单的 Web 应用程序(2)

一、实验目的及要求

该实验内容为通过 WEB 程序实现数据库的增、删、改和查询功能，并能使用 HttpSession 或 Cookie 实现会话状态数据管理，整个实验可以选择 J2EE/JSP 或.NET 实现。 该实验内容为通过 WEB 程序实现数据库的增、删、改和查询功能，并能使用 HttpSession 或 Cookie 实现会话状态数据管理。

二、总体设计

2.1 前端设计

①主页设计

主页前端要求布局类似学校门户，顶部有LOGO栏目，左侧是轮播图，右侧是登录表单。因此登录页面设计为左右布局，左边主要为LOGO栏目和轮播图，右边主要为登录表单和页脚。

②登录成功页面

由于实验要求可以简单设计登录成功页面，所以登录成功页面使用上下布局，header有三个按键，用于控制下面的显示内容；主体内容为修改密码表单/管理员查看用户信息表。

③错误页面

该页面仅用于用户请求错误显示，复用主页内容，右侧表单修改为一个登录按键。

2.2 前后端数据交互设计

使用json格式进行数据交互。

2.3 后端设计

①项目搭建

搭建maven项目，配置tomcat服务器，使用maven导入jar依赖包，基于MVC三层模型创建项目包结构（包含dao、filter、pojo、service、servlet、util等六个包），导入前端静态资源。

②数据库设计

需要数据库存储的信息只有用户信息，只需要在web数据库用户下建立一个USERS表即可。该表的字段有：id、tel、email、password、isadmin。

③编写实体类

使用OMR映射关系编写实体类，由于数据库中只有一个users表，所以需要编写一个users类与之对应。

④编写基础公共类

基础公共类包括数据库配置文件、数据库的公共类（提供对数据库的增删改查）、过滤器解决乱码问题。

⑤分模块实现登录、注册、修改密码功能、用户管理功能

每个模块都是基于MVC三层模型开发的，开发流程都一样。首先实现前端页面，即获得一个传输数据的表单；其次，从底层依次向上实现（Dao层、Service业务层、Servlet层）。

三、详细设计

3.1 前后端数据交互

前后端的数据交互分为以下两个步骤：前端发送一个请求request，请求包含消息内容；后台服务器响应response，响应包含消息内容。

目前主流的数据交互格式就是json格式。前端可以直接使用内置的JSON对象将json字符串转换为json对象/将json对象转换为json字符串；后端java使用fastjson包可以实现json对象与字符串的转换（虽然主流的依赖包是jackson）。

前后端的数据交互都是通过流传输的：前端把json字符串数据传输到request流中，后端java就读取这个输入流得到这些字符串消息；java响应数据时就是把json字符串写到输出流中，前端就得到了这些response的数据。

3.2 数据库的增删改查

  1. 加载驱动，获取数据库的连接
  2. 编写查询公共方法，使用preparedStatement预编译实现jdbc，实现对数据库的查询操作
  3. 编写查询update方法，使用preparedStatement预编译实现jdbc，实现对数据库的增删改操作

3.3 使用 HttpSession 或 Cookie 实现会话状态数据管理

由于该实验实现的是用户登录的功能，所以用户登录会将用户的信息放入session中，用户退出登录会移除该session。

六、遇到的问题及解决方案

6.1 乱码问题

前端收到后端的响应后，使用js修改前端页面内容出现乱码问题。原因是js与html的编码不一致。

解决方案：使用notepad++打开并设置编码为utf-8-Bom(标记字节顺序)

6.2 json字符串问题

使用XMLHttpRequest发送的json数据格式默认是XXX=XXX&XXX=XXX这样的，后端接收到后直接转换为json对象是会报错的。

解决方案：前端发送之前使用字符串拼接使其变成json字符串格式；后端接收到一个字符串数据后，使用字符串拼接方法过滤后再转换为json对象。

6.3 接收前端数据与响应数据问题

正常情况(type=button)下，前端send一个json字符串后，后端接受到的字符串应该是这样的：{”“：”“，”“：”“}，这种情况就可以直接使用JSONObject.fromObject(reqStr)将其转换为json对象；但是我在做的时候(type=submit,会使用默认的数据格式发送)，后端接受到的字符串是XXX=XXX&XXX=XXX这种类型，所以需要加上一些大括号和引号、冒号、逗号，使其能被json-jar包识别并转换为json对象。

Request流中的内容默认就是form表单中的name：value（键/值），其格式为XXX=XXX&XXX=XXX。所以在默认情况下读取HttpServletRequest流就是获取绑定的form表单中的name：value；所以可以直接使用req.getParameter(“LoginUserId”);读取对应的值。

把表单的类型设置为submit，后端java是可以直接使用req.getPar ameter(“XXX”)得到前端表单提交的数据的，也可以直接使用 resp.sendRedirect进行页面重定向或者 req.getRequestDispatcher().forward进行请求转发。

当表单类型不是submit时，需要我们手动的发送接收数据，即使用json格式进行数据交互。

6.4 HttpServletRequest的输入流只能读取一次

HttpServletRequest的输入流只能读取一次，所以在复用判断时读取一次，再在复用函数中再次读取就为空。

解决方案：复用判断前取出json对象，根据json对象的method的不同取值调用不同的复用函数，复用函数传递参数json。

6.5 一些细节问题

①这里的sql语句后面不能加逗号，否则jdbc就不认这个sql语句。

②如果没有注释那行输出，得到的值永远是false，因为本来的输出就是一行，next两次怎么还能查到数据呢

③无论怎么验证都是得到emailError错误
